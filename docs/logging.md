# Логування та обробка помилок (лабораторна 7)

## 1. Огляд рішень для Node.js / Express

Поширені бібліотеки: **Winston**, **Pino**, **Bunyan**. Обрано **Winston** через гнучкі **транспорти** (консоль + файли), інтеграцію з **winston-daily-rotate-file** (ротація за датою та обмеження розміру).

## 2. Типи помилок у проєкті STO

| Тип | Приклад | HTTP | Код у відповіді |
|-----|---------|------|-----------------|
| Валідація вхідних даних | Некоректне ім’я клієнта | 400 | `STO_VALIDATION` |
| Авторизація (демо) | Невірний пароль | 401 | `STO_AUTH` |
| БД / обмеження унікальності | Дубль телефону | 400 | `STO_DB` |
| Не знайдено | Клієнт / замовлення / маршрут | 404 | `STO_NOT_FOUND` / `STO_ROUTE_NOT_FOUND` |
| Необроблена помилка | Падіння поза `AppError` | 500 | `STO_INTERNAL` |

## 3. Рівні логування

Використовуються рівні npm/winston: **error**, **warn**, **info**, **verbose**, **debug**, **silly**. У коді застосовано:

- **error** — помилки `AppError` та необроблені винятки;
- **warn** — події з браузера (`/api/client-log`);
- **info** — старт процесу, успішні критичні операції (створення клієнта, замовлення), завершення HTTP-запиту (статус, тривалість).

## 4. Налаштування без перекомпіляції

Мінімальний рівень задається **змінною оточення** (Node не компілюється — достатньо перезапустити процес):

| Змінна | Приклад | Дія |
|--------|---------|-----|
| `LOG_LEVEL` | `debug`, `info`, `warn`, `error`, `silent` | Глобальний рівень логера |
| `LOG_DIR` | `D:\logs\sto` | Каталог файлів (за замовчуванням `./logs`) |
| `LOG_MAX_SIZE` | `10m` | Максимальний розмір одного файлу ротації |
| `LOG_MAX_FILES` | `14d` | Скільки днів/файлів зберігати |

Приклад (PowerShell):

```powershell
$env:LOG_LEVEL="debug"
npm start
```

У **тестах** (`NODE_ENV=test`) за замовчуванням рівень **`silent`**, щоб не засмічувати вивід Jest (для діагностики тестів можна виставити `LOG_LEVEL=info`).

## 5. Формат логів

- **Консоль:** час, рівень, сервіс, опційно `requestId`, повідомлення, JSON-метадані.
- **Файли:** JSON-рядки для подальшого парсингу (ELK тощо).

## 6. Унікальні ідентифікатори та контекст

- Кожен HTTP-запит отримує **`X-Request-Id`** (або приймається з заголовка клієнта).
- У JSON-помилках API: **`errorId`** (`crypto.randomUUID()`), **`code`**, **`requestId`**.
- У логах: ті самі поля + **контекст** операції (без зайвих персональних даних у проді).

## 7. Ротація логів

Реалізовано пакетом **winston-daily-rotate-file**:

- `logs/app-YYYY-MM-DD.log` — рівень **info** і вище;
- `logs/error-YYYY-MM-DD.log` — лише **error**;
- параметри **`maxSize`**, **`maxFiles`**, **`zippedArchive`**.

**Альтернатива зовнішніми засобами (Linux):** залишити запис у один файл і підключити **`logrotate`** (щоденна ротація, `copytruncate` або сигнал USR1 процесу — залежить від політики).

## 8. Локалізація

Повідомлення для користувача: **`locales/uk.json`**, **`locales/en.json`**. Мова: заголовок **`Accept-Language`** або **`?lang=en`**.

## 9. Сторінки для користувача

- **`public/errors/404.html`** — зрозумілий текст, посилання на головну, підказка звернутися до адміністратора.
- **`public/errors/500.html`** — без технічних деталей, інструкція оновити сторінку / підтримка.

Для шляхів **`/api/*`** помилки повертаються у **JSON** (для фронту та тестів).

## 10. Збір помилок з фронтенду

- **`POST /api/client-log`** — тіло: `message`, `source`, `url`, `line`, `stack` тощо; запис у лог рівня **warn**.
- У **`public/index.html`** підключені обробники **`error`** та **`unhandledrejection`**.

## 11. Приклад інформативної відповіді API (помилка)

```json
{
  "error": "Маршрут не знайдено",
  "errorId": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
  "code": "STO_ROUTE_NOT_FOUND",
  "requestId": "9f8e7d6c-5b4a-3210-fedc-ba9876543210"
}
```

## 12. Конфігураційні «файли»

Основна конфігурація логера — у **`logger.js`** + змінні оточення. Окремий YAML для лабораторної не введено, щоб не дублювати джерело істини; за потреби винесення у `.env` описано в `README.md`.

## 13. Скріншоти для звіту (рекомендація)

1. Консоль після `npm start` з рядками **info**.  
2. Вміст каталогу **`logs/`** після кількох запитів.  
3. Відповідь **404** у Postman/браузері для `/api/unknown` з полями **errorId** / **requestId**.  
4. Фрагмент **`error-*.log`** (JSON).
